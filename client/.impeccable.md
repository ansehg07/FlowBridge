# FlowBridge Design System

## Design Context

### Users
Professional accountants doing international financial analysis — comparing IFRS and US GAAP cash flow statements to derive cross-border business insights. They are domain experts who think in debits, credits, and standardized formats. They value accuracy above all and are comfortable with data-dense interfaces. They expect the tool to feel as authoritative as the data it presents. Trust and legibility are non-negotiable.

### Brand Personality
Modern · Precise · Mathematical

Emotional goal: the interface should make accountants feel **confident and in control** — like they are using a serious professional instrument, not a generic web form. The experience should feel sharp, not soft. Decisive, not decorative.

Reference: **Robinhood** — clean fintech confidence, strong typographic hierarchy, premium dark mode, numbers that feel authoritative. Data is the hero; UI recedes.

Anti-reference: Generic SaaS dashboards (bloated card grids, rainbow charts, excessive shadows). Disorganized, cluttered, or visually noisy layouts. Anything that looks like a template.

### Aesthetic Direction
- **Typography-first**: Large, confident numbers. Strong weight contrast between headings and body. Monospace for all financial figures. Inter for UI; JetBrains Mono for amounts.
- **Controlled color**: Muted base palette with precise accent colors. Green for confirmed/positive reconciliation, red for mismatches/errors, amber for warnings/flags. No decorative color.
- **Mathematical density**: Tables are primary UI surfaces — treat them like instruments, not afterthoughts. Tight, consistent spacing. Grid-aligned everything.
- **Premium restraint**: Subtle depth (not heavy shadows). Refined borders. Dark mode should feel like Bloomberg Terminal meets Robinhood — professional, not gloomy.
- **Confidence signals**: Circular confidence scores, clear status badges, and reconciliation indicators must read instantly. Hierarchy is about function, not decoration.

### Design Principles

1. **Data is the hero.** UI chrome should be invisible. Users came to read numbers, not admire the interface. Every decorative element must justify its existence.

2. **Precision in every pixel.** Consistent spacing scales, aligned columns, monospace numerals, uniform badge sizing. Sloppiness in layout implies sloppiness in data.

3. **Hierarchy through weight, not color.** Use typographic weight and size to establish importance. Reserve color strictly for semantic meaning (positive, negative, warning, error).

4. **Fintech confidence.** The product should feel like it was built by people who understand finance deeply. Sharp edges, tight spacing, authoritative tone — not rounded, friendly, or playful.

5. **Density with clarity.** Accountants need to see a lot at once. Don't hide data behind unnecessary accordions or progressive disclosure. Organize it spatially so everything important is reachable without hunting.
